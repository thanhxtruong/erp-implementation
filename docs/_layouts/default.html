Objective:

Develop and configure several quality modules in the ERP system to convert the current paper systems to electronic workflow-based systems.

Technical Wrap-up:
  • ERP System (IQMS)
  • Workflow and Electronic Signatures
  • Crystal Reports
  • Database Management (Oracle Database)
Systems Developed:
  • Deviation system for requesting and issuing temporary deviations.
  • Corrective Action Request (CAR) system for tracking and documenting customer complaints and corrective actions.
  • Material Review Board (MRB) system for review and approval of nonconforming product disposition.
  • Quick Inspection system for capturing inspection data.
Achievements:
  • Standardized the new processes with the use of electronic workflow.
  • Improved turn-around time for deviation approval with the use of broadcast workflow (Deviation system).
  • Improved accuracy in tracking expired deviations with automatic daily alert of upcoming expiration dates (Deviation system).
  • Improved timely response for MRB review and approval leading to improvement in the turn-around time for nonconforming product disposition (MRB system).
  • Integrated the Returned Material Authorization (RMA) process with the MRB process to streamline disposition of RMA materials (MRB system).
  • Improved data mining to generate KPI used in management review.
  • Improved overall timeliness (see metrics) in closing out complaints and corrective actions as well as on-time submission (CAR system).
Responsibilities:
  • System design, configuration, implementation, and training
  • Technical support and system maintenance

Challenge #1:

The (software) vendor-provided report takes up too much space resulting in unnecessary paper waste due to the report layout. How to re-design the layout to mimic the report in the current paper system that is more user-and-environmental-friendly?



Solution:

With the use of the “Format Groups with Multiple Columns” feature in Crystal Report allowing printing repeated columns horizontally, the improved layout as shown below has reduced the number of pages significantly (from 9 pages total to 3 pages total for the same record).



Challenge #2:

It is a requirement that Inspectors’ names, GageID’s (Measurement Equipment ID), and Inspection Notes are printed on the report. However, names and ID are mostly duplicate data entries and are only changed when multiple inspectors performed the inspection. Similarly, Inspection Notes are to be recorded per sample (row) only as necessary. How to print these data on the report without compromising the ability to use the “Format Groups with Multiple Columns” feature described in the previous page?

Below is a picture of what the table would look like with the additional columns. It is no longer possible to fit 3 repeated columns in the table.



Solution:

Use the “WhilePrintingRecords” function in Crystal Report to capture the data for each row printed and store them in a variable using String concatenation operator. For data with duplicate entries, use the “In” String operator in a conditional “if-then” statement to filter out duplicate records. The filtered and concatenated data is then printed in the group footer section of the report as shown below.
